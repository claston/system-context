## Summary

Describe what this PR changes in 2-4 bullets.

- Adds a production-ready `Dockerfile` for running the API service in containerized environments.
- Adds `.dockerignore` to reduce build context and avoid sending local artifacts to Docker builds.
- Adds `docs/docker-run-guide.md` with build/run instructions and env var usage.

## Why

This enables a consistent runtime for staging and deployment targets (such as Render) and reduces environment drift between local and hosted execution.

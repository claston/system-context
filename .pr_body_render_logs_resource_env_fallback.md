## Summary

- add optional Render logs fallback via `RENDER_LOGS_RESOURCE_ID` when no active `render-runtime` mapping exists
- keep existing DB mapping behavior as priority; env fallback is used only when mapping list is empty
- add test coverage for the env fallback path in connector dependency wiring
- document new env variables in local/staging example env files

## Why

Render logs API requires a `resource` id (`srv-*`). In environments where target mapping is not yet configured in DB, logs analysis fails. This change enables a temporary, explicit env-based fallback for testing and staged rollout.
